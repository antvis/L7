## 重构原则

1. **行为等价优先**：每个阶段先保证现有 spec 通过，再考虑行为优化
2. **对外 API 完全向后兼容**：内部重构用 delegate / getter 别名 / re-export 转发，`ISource` 接口与本包公共导出不动
3. **小步快跑**：每个阶段拆成独立可 merge 的 PR，配 changeset
4. **先清理后优化**：阶段 0 做零风险清理，阶段 1+ 才动结构
5. **文档先行**：每步重构先在 `PROGRESS.md` 登记，再写代码

## 如何继续

后续接手时：

1. 读 `PLAN.md` 了解整体路线
2. 读 `PROGRESS.md` 顶部「下一步」字段，知道当前应该做什么
3. 读 `BACKLOG.md` 了解已知遗留问题，避免重复发现
4. 完成一步后，更新 `PROGRESS.md` 与 `BACKLOG.md`
